# 📘 **DOC\_STYLE.md**

**Guía oficial de estilo de documentación del ecosistema LINCAT / CNC‑CAT**

# **1. Propósito**

Esta guía define **cómo debe escribirse toda la documentación** del ecosistema:

- clara

- industrial

- consistente

- modular

- profesional

- mantenible

Es el estándar que garantiza que todos los documentos tengan la misma calidad y estilo.

# ⭐ 2. Principios fundamentales

### **2.1 Claridad**

- Frases cortas

- Conceptos directos

- Nada de ambigüedad

- Nada de adornos innecesarios

### **2.2 Modularidad**

- Un documento = un tema

- Un archivo = una responsabilidad

- Nada de mezclar conceptos

### **2.3 Consistencia**

- Mismos títulos

- Mismo orden

- Mismo estilo

- Mismo tono

### **2.4 Profesionalidad**

- Estilo industrial

- Terminología técnica correcta

- Diagramas cuando sean necesarios

# ⭐ 3. Estructura estándar de cada documento

Cada documento debe seguir esta estructura:

Código

```
`\# Título principal`

`\#\# 1. Propósito`

`\#\# 2. Arquitectura / Concepto / Diseño`

`\#\# 3. Componentes`

`\#\# 4. Integración`

`\#\# 5. Ejemplos (si aplica)`

`\#\# 6. Conclusión`
```

# ⭐ 4. Estilo de escritura

### **4.1 Tono**

- Técnico

- Profesional

- Directo

- Sin opiniones personales

### **4.2 Lenguaje**

- Español neutro

- Terminología industrial

- Evitar coloquialismos

### **4.3 Formato**

- Markdown puro

- Listas claras

- Diagramas ASCII cuando sea útil

- Código en bloques triple backtick

# ⭐ 5. Reglas de formato

### **5.1 Títulos**

- `\#` para título principal

- `\#\#` para secciones

- `\#\#\#` para subsecciones

### **5.2 Listas**

- Usar `-` o `\*`

- No mezclar estilos

### **5.3 Código**

Código

```
`\`\`\`python`

`\# ejemplo`
```

Código


`\#\#\# \*\*5.4 Diagramas ASCII\*\*`

`Permitidos y recomendados:`


\[ UI \] → \[ Motion API \] → \[ Core \]
